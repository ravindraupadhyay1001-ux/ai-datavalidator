📅 Any path/query field supports date tokens for daily feeds: {YYYYMMDD}, {D-1} (yesterday), {D+1}, {YYYYMMDD-2}, {YYYY}, or {strftime:%Y-%m-%d} — e.g. "remote_path": "/feeds/trades_{YYYYMMDD}.csv".

🔒 Any password/secret field supports ${ENV:VAR_NAME} to pull a credential from the server environment instead of storing it here — e.g. "password": "${ENV:PROD_DB_PASSWORD}".

↻ SQL/BigQuery sources support incremental fetch: set incremental_column (e.g. updated_at) and since (a value or date token like {YYYYMMDD-1}) to pull only new/changed rows on each run.

⚠️ Table mode defaults to a 100,000-row cap to protect against accidentally pulling an entire large production table into memory. Raise row_limit or set it to 0 for unlimited if you really need the whole table.
